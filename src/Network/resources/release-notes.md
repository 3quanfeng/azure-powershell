# Az.Network Release Notes

## Version 0.1.0-preview
### What's New
- A majority of the removed cmdlets were creating in-memory objects and did not communicate with Azure. This information is now part of the parameters of the cmdlets that do communicate with Azure. Some of these parameters are complex types. These complex type parameters will allow you to provide a hashtable with the appropriate data instead of creating the complex type explicitly. Complex type documentation can be found in the Notes section of the cmdlet help.
- In several cmdlets originally, they would have taken an entire object of a child/sibling-resource. Now, those cmdlets will take a reference to that child/sibling-resource instead of the entire content of that resource. This likely adjusts workflows so that creating the child/sibling-resource needs to come first, and then supplied to the parent/sibling-resource. This added consistency across the cmdlets in the module, and follows conventions of other Azure services/modules.
- The names of the cmdlets/parameters have been adjusted to be more consistent. If there are standard short-names/abbreviations for industry terms, these are used instead to help keep cmdlet/parameter names shorter. For example, NetworkSecurityGroup is now Nsg and VirtualNetwork is now Vnet. If there was an original name for the cmdlet/parameter, aliases were added back to those original names. For example, New-AzVirtualNetwork still exists, but as an alias to New-AzVnet.

### Future Enhancements
- A set of convenience cmdlets previously existed. These are not part of this preview, but would be implemented in the future:
  - Get-AzNetworkWatcherSecurityGroupView
  - Update-AzVirtualHub
  - Update-AzVirtualWan
  - Update-AzVpnGateway
  - Update-AzVpnSite
